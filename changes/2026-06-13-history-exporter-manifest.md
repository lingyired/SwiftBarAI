# 2026-06-13: History export zip includes MANIFEST.json with provider + version

- **Type:** feat
- **Scope:**
  `menubar01/UI/Generator History/GeneratorHistoryExporter.swift`,
  `menubar01Tests/GeneratorHistoryExporterTests.swift`
- **Author(s):** Trae AI
- **Commit(s):** _pending_
- **Status:** in-progress

## Summary

The audit-log zip produced by `GeneratorHistoryExporter` now
carries a small `MANIFEST.json` at the archive root, alongside
`<promptId>/request.txt` / `response.json` / `menu.json`. The
manifest records the menubar01 version + build, the export
timestamp, the entry count, and a `provider` string derived
from the entry's `endpointHost` (and `model` as the fallback).
Support tooling can now attribute a shared history bundle to
the model + endpoint that produced it without having to crack
open `response.json`.

## Motivation

`AIGeneratorHistoryEntry` learned to record `endpointHost` (for
remote runs) in
[`2026-06-13-history-records-endpoint-host.md`](2026-06-13-history-records-endpoint-host.md),
and the M5 history sheet now shows
"Generated by `<model>` at `<host>`" in the detail pane (a
follow-up rendered by `GeneratorHistorySheet.generatedBySection`).
The "Export…" zip still bundled the per-entry audit log but
carried no top-level metadata: a shared zip landed in support's
inbox with no easy way to know which model produced it, which
endpoint served it, or which build of menubar01 exported it.
The fix is a single small JSON file at the zip root.

## Changes

### `menubar01/UI/Generator History/GeneratorHistoryExporter.swift`

- New `static let manifestFilename = "MANIFEST.json"` (so the
  test bundle and any future tooling can reference the
  constant).
- `runZip(sourceDirectory:destination:entry:)` gains a new
  trailing `entry: AIGeneratorHistoryEntry? = nil` parameter.
  When non-nil, `writeManifest(into:entryCount:provider:)`
  is invoked before the zip is created; the resulting
  `MANIFEST.json` lives next to the entry's audit log in
  `sourceDirectory`, which becomes the archive root after
  `zip -r .` so the manifest surfaces at the top of the
  extracted bundle.
- `exportEntry(_:store:)` (the `NSSavePanel` entry point) now
  passes the entry through to `runZip(...)` so every UI-driven
  export automatically gets the manifest. The lower-level
  `runZip(sourceDirectory:destination:)` (without the new
  `entry:`) keeps its previous behaviour, which is what the
  pre-existing test cases call.
- New private `writeManifest(into:entryCount:provider:) throws`
  that serialises a `HistoryExportManifest` value with
  `[.prettyPrinted, .sortedKeys, .withoutEscapingSlashes]` and
  `dateEncodingStrategy = .iso8601` (matching the rest of the
  project's JSON output) and writes it atomically to
  `sourceDirectory/MANIFEST.json`. On failure the `runZip`
  caller surfaces a `.zipFailed(reason:)` so the
  `GeneratorHistorySheet` "Export…" alert still routes
  through a single error path.
- New private `providerString(for:)` that maps
  `entry.endpointHost` / `entry.model` to one of three stable
  strings:
  - `endpointHost` non-empty → `"remote:<host>"`
  - `endpointHost` nil, `model` non-empty → `"local"`
  - both empty → `"unknown"`
- New private `HistoryExportManifest: Codable` value type with
  the five fields (`appVersion`, `appBuild`, `exportedAt`,
  `entryCount`, `provider`). `appVersion` /
  `appBuild` are read from `Bundle.main.infoDictionary`
  with `"unknown"` fallbacks; `exportedAt` is a `Date` that
  the encoder's `dateEncodingStrategy` formats as ISO-8601.

### `menubar01Tests/GeneratorHistoryExporterTests.swift`

- The `makeEntry(...)` test helper gains
  `endpointHost: String? = nil` and `model: String = "gpt-4o-mini"`
  parameters so manifest-related tests can pin each entry's
  provider derivation inputs.
- New private `extractZip(_:to:)` helper that runs
  `/usr/bin/unzip -o <zip> -d <temp>` and returns the
  extraction directory, so the new tests can read files
  directly out of the archive instead of scraping the
  human-readable `unzip -l` listing.
- 4 new Swift-Testing tests in `GeneratorHistoryExporterTests`:
  - `testExportEntry_zipContainsManifest` — exports an entry
    with `endpointHost: "api.openai.com"`, unzips, asserts
    `MANIFEST.json` exists at the archive root.
  - `testExportEntry_manifestHasCorrectProvider` — same
    entry, asserts the manifest's `provider` field is
    `"remote:api.openai.com"`.
  - `testExportEntry_manifestProviderIsLocalForNilHost` —
    `endpointHost: nil`, `model: "local-model"`, asserts
    `provider == "local"`.
  - `testExportEntry_manifestHasCorrectEntryCount` — single
    entry, asserts `entryCount == 1`. The multi-entry
    export path is a follow-up; when it lands it will
    pass the actual count to `writeManifest(into:entryCount:provider:)`.

## Manifest schema

`MANIFEST.json` is pretty-printed, sorted-key JSON. The
current shape:

```json
{
  "appBuild": "170",
  "appVersion": "1.2.3",
  "entryCount": 1,
  "exportedAt": "2026-06-13T12:34:56Z",
  "provider": "remote:api.openai.com"
}
```

Field semantics:

- `appVersion`: `CFBundleShortVersionString` from
  `Bundle.main.infoDictionary`, or `"unknown"`.
- `appBuild`: `CFBundleVersion` from
  `Bundle.main.infoDictionary`, or `"unknown"`.
- `exportedAt`: ISO-8601 string of `Date()` at export time.
- `entryCount`: 1 for the single-entry export path.
  Future multi-entry exports will set this to the actual
  count.
- `provider`: one of
  `"remote:<endpointHost>"` (non-nil host),
  `"local"` (nil host + non-empty model),
  `"unknown"` (both empty).

## Impact

- **User-visible.** Any zip produced by the "Export…" button
  in `GeneratorHistorySheet` now includes a `MANIFEST.json`
  at the root. The user does not interact with the manifest
  directly; the change is purely a support-tooling
  affordance so a shared bundle is self-describing.
- **Internal.** One new optional parameter
  (`entry: AIGeneratorHistoryEntry? = nil`) on the existing
  `GeneratorHistoryExporter.runZip(...)` API, plus two new
  private helpers (`writeManifest(into:entryCount:provider:)`,
  `providerString(for:)`) and one new private value type
  (`HistoryExportManifest`). No public type signatures
  changed; `exportEntry(_:store:)` keeps the same
  `GeneratorHistoryExportResult` enum result.
- **Security.** The manifest carries the same redacted host
  that `AIGeneratorHistoryEntry.endpointHost` already
  exposes, plus the entry count and a build stamp. No
  apiKey, full URL, or per-entry content is duplicated
  into the manifest.
