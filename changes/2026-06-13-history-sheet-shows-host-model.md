# 2026-06-13: history detail view shows model and endpoint host

- **Type:** feat
- **Scope:**
  `menubar01/UI/Generator History/GeneratorHistorySheet.swift`,
  `menubar01Tests/GeneratorHistoryViewModelTests.swift`
- **Author(s):** Trae AI
- **Commit(s):** 7eee0f2
- **Status:** done

## Summary

The M5 generator history detail view now shows "Generated by
`<model>` at `<host>`" as a small secondary-text section
between the metadata header and the request body, so the user
can tell at a glance whether a recorded run was produced by
a remote provider (e.g. `gpt-4o` at `api.openai.com`) or a
local / mock run (e.g. `gpt-4o` at `local model`). The new
section reads from the same `AIGeneratorHistoryEntry` value
type the previous
[`2026-06-13-history-records-endpoint-host.md`](2026-06-13-history-records-endpoint-host.md)
round wired up — no new public API on the view model, no
schema change, no new test fixture.

## Motivation

`AIGeneratorHistoryEntry` has been recording both `model`
(always) and `endpointHost` (when a remote provider dialed
out) since the previous commit, but the SwiftUI sheet only
exposed `model` (in a "label: value" row inside the
metadata header). `endpointHost` was never rendered, so
remote runs were visually indistinguishable from local /
mock runs in the history detail view. The
"Generated by `<model>` at `<host>`" framing called out in
the previous record's motivation could not be realised
without this UI change.

## Changes

### `menubar01/UI/Generator History/GeneratorHistorySheet.swift`

- New private `generatedBySection(for:)` function. Renders a
  small `VStack` with a `"Generated by"` subhead followed by
  two `HStack` rows in the same label/value style as
  `detailHeader`:
  - `Model: <entry.model>` — always shown, monospaced for
    parity with the existing model row.
  - `From: <entry.endpointHost ?? "local model">` — shown
    with a sensible fallback for Mock / Local / LocalEcho
    runs and for any pre-M5+ entry whose `response.json`
    predates the `endpointHost` key (the existing
    `decodeIfPresent(_:forKey:)` round-trip in
    `AIGeneratorHistoryEntry.init(from:)` keeps those
    decodes clean).
- `detailPane` now inserts the new section between
  `detailHeader` and `requestSection`, framed by the same
  `Divider()` separators as the other sections.
- No change to `detailHeader` (the pre-existing `promptId` /
  `createdAt` / `model` rows stay put, so the dev-facing
  metadata view is unchanged) and no change to the
  destructive buttons in the footer.
- No public-API change. `AIGeneratorHistoryEntry.model`
  and `AIGeneratorHistoryEntry.endpointHost` were already
  `public let` properties, so the SwiftUI sheet reads them
  off `viewModel.selectedEntry` directly.

### `menubar01Tests/GeneratorHistoryViewModelTests.swift`

- `makeTestEntry(...)` gains two new defaulted parameters:
  - `model: String = "gpt-4o-mini"` — keeps the existing
    call sites compiling unchanged.
  - `endpointHost: String? = nil` — forwarded to
    `AIGeneratorHistoryEntry.init(...)` so the new tests
    can exercise the remote / local codepath.
- New `GeneratorHistoryViewModelSelectedEntryExposesModelAndHostTests`
  suite with 3 tests:
  - `testSelectedEntry_exposesModel` — records an entry
    with `model: "gpt-4o"`, sets the view-model
    selection, and asserts
    `viewModel.selectedEntry?.model == "gpt-4o"`. Pins the
    contract that the SwiftUI sheet reads the model from
    `selectedEntry` (not from a custom accessor).
  - `testSelectedEntry_exposesEndpointHost` — same shape,
    with `endpointHost: "api.openai.com"`, asserting the
    string flows through the reload → selection pipeline
    verbatim.
  - `testSelectedEntry_endpointHostIsNilForLocal` — same
    shape with `endpointHost: nil` (the default), asserting
    the view model surfaces `nil` so the sheet's
    `"local model"` fallback is reached.

## Impact

- **User-visible.** The M5 generator history detail view
  gains a small "Generated by" section near the top of the
  detail pane. The section uses `.subheadline` (header) +
  `.caption` (rows) styling and `.secondary` foreground, so
  it does not compete with the primary "Request" /
  `manifest.json` / "Entry script" sections. No other UI
  surfaces change.
- **Internal.** No new public API on the `menubar01`
  module. The only change to `AIGeneratorHistoryEntry`
  reachability is that the SwiftUI sheet now reads
  `entry.endpointHost` (which it could already do via
  `@testable import` in the test bundle). The view model's
  public API is unchanged.
- **Backward compatibility.** Older `response.json` files
  (no `endpointHost` key) decode to `endpointHost == nil`
  and render as `From: local model`, exactly as the
  pre-existing fallback already promises.
